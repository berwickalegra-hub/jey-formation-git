## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → backend (route handlers, Wave 1) | Untrusted password input flows through banned-passwords + HIBP gates BEFORE bcrypt; lockout-store records failed attempts |
| backend → external (HIBP API) | Trust boundary on outbound; opt-in via env; fail-open with warn; 2s timeout |
| backend → Redis | Lockout + refresh-lock keyed on email/userId; lower-cased to prevent case-mutation bypass |

## STRIDE Threat Register

| Threat ID | Severity | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|----------|-----------|-------------|-----------------|
