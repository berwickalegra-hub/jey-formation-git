Land the change-password route — the only authenticated mutation in Phase 1 — covering AUTH-09. ## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → backend | Authenticated session (cookie) + CSRF header; double-submit pattern enforced |
| backend → DB | Single user.update transaction touching passwordHash + tokenVersion |

## STRIDE Threat Register

| Threat ID | Severity | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|----------|-----------|-------------|-----------------|
| T-1-02 | HIGH | Tampering | change-password (CSRF) | mitigate | verifyCsrf MANDATORY first call (D-02); 403 on missing/mismatched CSRF header |
| T-1-02 | HIGH | Elevation of Privilege | change-password (session takeover by stolen-cookie attacker) | mitigate | tokenVersion: { increment: 1 } in same DB write — invalidates ALL other sessions (D-19); attacker's stolen cookie fails next requireAuth call |
| T-1-02 | HIGH | Tampering | change-password (current-password bypass) | mitigate | verifyPassword(currentPassword) BEFORE update; bcrypt cost 12; rejects wrong password with INVALID_CREDENTIALS |
| T-1-02 | HIGH | Tampering | change-password (weak new password) | mitigate | Password policy gates: length ≥ 10 + banned-list + HIBP (opt-in) — all checked BEFORE DB read so weak-password attempts don't even hit prisma |
| T-1-02 | HIGH | Availability | change-password (Pitfall 9 — user logs themselves out) | mitigate | After tokenVersion bump, IMMEDIATELY mint new access + refresh tokens with bumped tokenVersion and call setAuthCookies + setCsrfCookie; current browser stays logged in |
| T-1-04 | MEDIUM | Information Disclosure | change-password logs | mitigate | log.info logs userId only — never currentPassword/newPassword/email plaintext; redaction via observability/log.ts |
| T-1-05 | MEDIUM | Repudiation | change-password | mitigate | requestId injected via Phase 0 ALS + x-request-id response header |
| T-1-08 | LOW | Misconfiguration | AUTH_PASSWORD_MIN_LENGTH + PASSWORD_HIBP_CHECK env vars | mitigate | Defaults baked in (10, off); ROADMAP Plan 05 documents in .env.example |
</threat_model>
